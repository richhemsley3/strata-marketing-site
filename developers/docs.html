API Reference

Complete reference for the Strata REST API. All endpoints use JSON request and response bodies. Authenticate with Bearer tokens and manage your data sources, classifications, and policies programmatically.

Authentication

The Strata API uses Bearer token authentication. Include your API key in the Authorization header of every request. API keys can be created and managed from the Settings > API Keys page in your Strata dashboard.

Authorization: Bearer sk_live_your_api_key_here

Authentication Methods

Strata supports three authentication methods depending on your integration needs:

Method Use Case Details
Bearer Token Server-to-server API calls Pass sk_live_... in the Authorization header. Recommended for backend integrations.
API Key Quick prototyping Pass key as a query parameter: ?api_key=sk_live_.... Not recommended for production.
OAuth 2.0 Third-party applications Standard authorization code flow. Contact support to register your OAuth application.

All API requests must be made over HTTPS. Requests made over HTTP will be rejected. API keys carry significant privileges -- keep them secure and never expose them in client-side code or public repositories.

Rate Limiting

The Strata API enforces rate limits to ensure fair usage and platform stability. Rate limits are applied per API key and reset every 60 seconds. When you exceed a rate limit, the API returns a 429 Too Many Requests response.

Endpoint Rate Limit Burst
GET endpoints 200 requests / min 50 concurrent
POST endpoints 60 requests / min 10 concurrent
PUT / PATCH endpoints 60 requests / min 10 concurrent
DELETE endpoints 30 requests / min 5 concurrent
/v1/classifications (POST) 20 requests / min 3 concurrent

Rate limit headers are included in every response: X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset. Use these to implement backoff strategies in your integrations.

Errors

Strata uses conventional HTTP response codes to indicate the success or failure of an API request. Codes in the 2xx range indicate success, codes in the 4xx range indicate a client error, and codes in the 5xx range indicate a server error.

Code Status Description
400 Bad Request The request body is malformed or missing required parameters.
401 Unauthorized No valid API key was provided in the Authorization header.
403 Forbidden The API key does not have permission to perform the requested action.
404 Not Found The requested resource does not exist or has been deleted.
429 Too Many Requests Rate limit exceeded. Wait and retry after the X-RateLimit-Reset interval.
500 Internal Server Error An unexpected error occurred on the Strata server. Contact support if this persists.

List all sources

GET /v1/sources Try it

Returns a paginated list of all data sources connected to your organization. Results can be filtered by type and are sorted by creation date, with the most recent sources appearing first.

Query Parameters

Parameter Type Description
limit integer optional Maximum number of sources to return. Default: 20, max: 100.
offset integer optional Number of records to skip for pagination. Default: 0.
type string optional Filter by source type. One of: postgresql, mysql, snowflake, s3, bigquery.

Response

Returns an object with a data array of source objects and pagination metadata. Each source includes its unique ID, display name, connection type, region, status, and timestamps.

Create a source

POST /v1/sources Try it

Registers a new data source with Strata. Once created, the platform will begin an initial scan to discover and classify sensitive data within the source.

Request Body Parameters

Parameter Type Description
name string required A human-readable display name for the data source.
type string required The database or storage type. One of: postgresql, mysql, snowflake, s3, bigquery.
connection_string string required Encrypted connection URI or ARN for the data source.
region string optional Cloud region where the source is hosted. Example: us-east-1.

Delete a source

DELETE /v1/sources/{id} Try it

Permanently removes a data source and all associated scan history from Strata. This action cannot be undone. Active policies referencing this source will be automatically disabled.

Path Parameters

Parameter Type Description
id string required The unique identifier of the data source to delete. Starts with src_.

List classifications

GET /v1/classifications Try it

Returns a paginated list of all data classification rules configured for your organization. Classifications define how Strata identifies and labels sensitive data patterns such as PII, PHI, financial data, and custom patterns.

Query Parameters

Parameter Type Description
limit integer optional Maximum number of classifications to return. Default: 20, max: 100.
offset integer optional Number of records to skip for pagination. Default: 0.
category string optional Filter by category. One of: pii, phi, financial, credentials, custom.
severity string optional Filter by severity level. One of: critical, high, medium, low.

Response

Returns an object with a data array of classification objects. Each classification includes its ID, label, category, severity level, detection pattern, and timestamps.

Create a classification

POST /v1/classifications Try it

Creates a new custom classification rule. Strata will apply this rule during subsequent scans to identify matching data patterns across all connected sources.

Request Body Parameters

Parameter Type Description
label string required Human-readable name for the classification (e.g., "Social Security Number").
category string required Classification category. One of: pii, phi, financial, credentials, custom.
severity string required Severity level. One of: critical, high, medium, low.
pattern string required Regex pattern used to detect this data type (e.g., \d{3}-\d{2}-\d{4}).
description string optional Additional context about what this classification identifies.
enabled boolean optional Whether the classification is active. Default: true.

List policies

GET /v1/policies Try it

Returns a paginated list of all data governance policies. Policies define the rules and actions that Strata enforces when sensitive data is detected, including masking, alerting, and access control.

Query Parameters

Parameter Type Description
limit integer optional Maximum number of policies to return. Default: 20, max: 100.
offset integer optional Number of records to skip for pagination. Default: 0.
status string optional Filter by status. One of: active, inactive, draft.
action string optional Filter by enforcement action. One of: mask, alert, block, log.

Response

Returns an object with a data array of policy objects. Each policy includes its ID, name, associated classifications, enforcement action, scope, and timestamps.

Create a policy

POST /v1/policies Try it

Creates a new governance policy. Policies bind classification rules to enforcement actions and can be scoped to specific data sources or applied organization-wide.

Request Body Parameters

Parameter Type Description
name string required Human-readable name for the policy (e.g., "Mask SSN in Production").
classification_ids array required Array of classification IDs this policy applies to.
action string required Enforcement action. One of: mask, alert, block, log.
source_ids array optional Array of source IDs to scope this policy to. Omit for org-wide enforcement.
description string optional Description of the policy's purpose and scope.
enabled boolean optional Whether the policy is active on creation. Default: true.

Update a policy

PUT /v1/policies/{id} Try it

Updates an existing policy. All fields are optional -- only the provided fields will be updated. Changes take effect immediately for active policies.

Path Parameters

Parameter Type Description
id string required The unique identifier of the policy. Starts with pol_.

Request Body Parameters

Parameter Type Description
name string optional Updated display name for the policy.
action string optional Updated enforcement action.
enabled boolean optional Enable or disable the policy.
classification_ids array optional Updated list of classification IDs.
source_ids array optional Updated list of source IDs for scope.

Delete a policy

DELETE /v1/policies/{id} Try it

Permanently deletes a governance policy. This action cannot be undone. Data sources previously governed by this policy will no longer have its enforcement action applied.

Path Parameters

Parameter Type Description
id string required The unique identifier of the policy to delete. Starts with pol_.
